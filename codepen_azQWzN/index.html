The Basics of the Web and HTML

During this stage of the nanodegree I learned a great deal about the parts that make up the web, as well as how to build the main type of document on the web. Here are the main takeaways from Stage 0.

Major Pieces of the Web

-http: The main protocol of the web; used to communicate between you, servers, and browsers
-Servers: Computers that host the files that make up the web
-Internet: The world's largest computer network
-Browser: The program that runs on the computer to display files found on the web

HTML

The main document found on the web, HTML, stands for HyperText Markup Language. HTML is made up of the following:

-Text Content: What you see
-Markup: What it looks like
-References to other documents (images, videos)
-Links to other pages

HTML Markup

Made up of tags, elements, and attributes.

-Tag: Represented by angled brackets; opening and closing tags; void tags
-Element: Everything within a set of opening and closing tags
-Attribute: A property of an HTML element

Tags and Elements

Opening tags come before the content, while closing tags go after the content. Each set of opening tags, content, and closing tags make up an element. There are also void tags, which do not require closing tags to make up an element. Below is a list of tags that were discussed in this lesson:

b: Bold font
em: Italic font
br: Break; void tag; allows to type text on multiple lines (inline)
p: Paragraph; also allows to type text on multiple lines; requires closing tag (block)

While the br and p tags perform similar functions (both make text appear on multiple lines), there are a couple differences that are noteworthy. Unlike the p tag, the br tag does not require a closing tag. This is known as a void tag. In addition, br is inline, while p is block. Think of block text as having an imaginary box around it. This results in an extra space that a single br tag does not contain.

Attributes

Attributes are properties of HTML elements. These allow for content such as links, images, and videos to show up on a webpage. Here are some examples:

anchor: represented as the letter a
image: represented as img; void tag

Attributes require an attribute name, a value, and text to show what the attribute will look like to the internet user.
